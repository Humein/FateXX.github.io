项目对于重复功能抽取的可做以下处理

基类

  • 抽象出一个基类； A ,B,C 继承它

    缺点：

  • 可读性很差，例如，对于 A/B/C 来说，你看不出来任何feature的行为, 个类里面没有任何东西能告诉你它需要feature。这样表意不明确，因为别处可能会随机存在一个feature，你甚至不知道它是从哪里来的

  • 随着业务增加，基类会越来越臃肿。

分类

  • 对他们的基类比如UIView进行扩展, 避免不同类型的feature都融合在一个类中

    缺点:

  • 对于不同的类型Object就无法通用

协议

  • 功能具有插拔性

  • 适用场景广泛

  • 避免一个类其实只要一个功能情况下，也要继承着多个功能模块

    缺点:

    OC的协议没有默认实现

了解不同的方案优缺点，可以应对开发中不同的case。

利用Hexo搭建博客

  1. 新建一个空文件夹HexoBlog

    • 这里放着Hexo 相关文件以及你后续新建的blog

  2. 使用Hexo生成博客，初始化一个博客

    • 进入HexoBlog文件夹：cd HexoBlog
    • 初始化：hexo init
    • 启动hexo：hexo s
      • 如果提示4000端口被占用(之前搭建过)，解决方法如下
        • sudo lsof -i:4000，查看被占用进程的pid
        • 再输sudo kill -9 pid （pid为上一步相应查看到的）
        • 然后再继续hexo s，则成功
    • 浏览器输入：http://localhost:4000 可查看到默认生成了一篇博文

  3. 新建一篇博文

    • ctrl+c断开刚刚的链接，然后新建一篇博文：hexo n "发布第一篇个人博客"。生成出来的就是一个MD文件
    • 清理：hexo clean
    • 生成：hexo g
    • 启动：hexo s
    • 更新MD文件后，然后刷新网页就行了。
  4. 编辑 MarkDown(Blog)
    • Balabala…..

部署至github

  1. 在GitHub上新创建一个repository，仓库名必须为：yourNick.github.io:

  2. 命令行下，ctrl+c，退出命令，仍然在HexoBlog目录下，安装git部署插件：npm install hexo-deployer-git –save`

  4. 回到命令行部署到远端

    • hexo d

      • 如果：发现https://humein.github.io/ 和本地的不一样；再次hexo clean -> hexo d 就行了
  5. Blog 更新
