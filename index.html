利用Macbook+移动硬盘实现NAS的效果

预备设备和知识

移动硬盘

现在买到的希捷移动硬盘都会带有自动休眠功能。正好配合MacBook的节能 完美实现用之则行,舍之则藏😄。

MacBook

主要利用MacBook两个功能

  • 使硬盘进入睡眠：在“电池”或“电源适配器”面板中选择“如果可能，使硬盘进入睡眠”选项。
  • 允许 Mac 短暂唤醒方便用户访问共享的服务（如果适用）：请选择任一可用的“唤醒以…”选项，例如“唤醒以供网络访问”。
    • Mac 睡眠时共享其资源：如果您正在共享电脑资源，例如共享打印机或“音乐”播放列表，则可以设定电脑在其他用户访问这些资源时从睡眠中唤醒。若要使用这个功能，您需要支持 802.11n 并安装了最新固件的 Apple 无线设备。另外，如果使用的是便携式 Mac，应使用电源适配器为其充电。
      • 选择此“唤醒以供网络访问”复选框后，您的 Mac 将在以下情况下唤醒：
        • 当其他电脑的用户访问电脑的共享资源时。
        • 在您的 Mac 每隔一段时间向网络上的其他电脑通知其资源可用时。
    • 实际应用发现同一Wi-Fi下就算你不去访问这些共享资源，MacBook也会从休眠中唤醒，具体原因不明。我猜测：
      • MacBook自身应用去唤醒的
      • 同一Wi-Fi下其他设备误触发了MacBook，然后被唤醒。
        • 这个可以使用一个干净的私有Wi-Fi去尝试解决：只有MacBook以及你目前手持的设备，如果结束使用还需要将设备切换到公共Wi-Fi。

如果家中有闲置的MacBook是最好的了-不用频繁去移动设备。

quote: 设定 Mac 的睡眠和唤醒时间

Wi-Fi

用一个路由器分成两个Wi-Fi

  • 公共Wi-Fi - 5G
  • 私有Wi-Fi - 2.4G

注：

  • 局域网中的数据传输速度和路由器最大支持带宽和信号强度有关
  • 5GHz的传输速率是5GHz要比2.4GHz要快。
  • 第五代802.11ac协议的5GHz Wi-Fi 频段，80MHz的频宽下能达到433Mbps的传输速度，160MHz的频宽下更能达到866Mbps的传输速度，相比最高速率300Mbps的2.4GHz的传输速率有了大大的提升
  • 5GHz Wi-Fi也有缺点，它的缺点在于传输距离和穿越障碍物能力.频率高波长短的5G信号，覆盖范围相对较小，穿过障碍物的能力不如2.4GHz。
  • 一般2.4GHz Wi-Fi在室内最大能达到70米覆盖，室外最大能达到250米覆盖.而5GHz Wi-Fi在室内最大只能达到35米覆盖。

项目对于重复功能抽取的可做以下处理

基类

  • 抽象出一个基类； A ,B,C 继承它

    缺点：

  • 可读性很差，例如，对于 A/B/C 来说，你看不出来任何feature的行为, 个类里面没有任何东西能告诉你它需要feature。这样表意不明确，因为别处可能会随机存在一个feature，你甚至不知道它是从哪里来的

  • 随着业务增加，基类会越来越臃肿。

分类

  • 对他们的基类比如UIView进行扩展, 避免不同类型的feature都融合在一个类中

    缺点:

  • 对于不同的类型Object就无法通用

协议

  • 功能具有插拔性

  • 适用场景广泛

  • 避免一个类其实只要一个功能情况下，也要继承着多个功能模块

    缺点:

    OC的协议没有默认实现

了解不同的方案优缺点，可以应对开发中不同的case。

利用Hexo搭建博客

  1. 新建一个空文件夹HexoBlog

    • 这里放着Hexo 相关文件以及你后续新建的blog

  2. 使用Hexo生成博客，初始化一个博客

    • 进入HexoBlog文件夹：cd HexoBlog
    • 初始化：hexo init
    • 启动hexo：hexo s
      • 如果提示4000端口被占用(之前搭建过)，解决方法如下
        • sudo lsof -i:4000，查看被占用进程的pid
        • 再输sudo kill -9 pid （pid为上一步相应查看到的）
        • 然后再继续hexo s，则成功
    • 浏览器输入：http://localhost:4000 可查看到默认生成了一篇博文

  3. 新建一篇博文

    • ctrl+c断开刚刚的链接，然后新建一篇博文：hexo n "发布第一篇个人博客"。生成出来的就是一个MD文件
    • 清理：hexo clean
    • 生成：hexo g
    • 启动：hexo s
    • 更新MD文件后，然后刷新网页就行了。
  4. 编辑 MarkDown(Blog)
    • Balabala…..

部署至github

  1. 在GitHub上新创建一个repository，仓库名必须为：yourNick.github.io:

  2. 命令行下，ctrl+c，退出命令，仍然在HexoBlog目录下，安装git部署插件：npm install hexo-deployer-git –save`

  3. 更改HexoBlog文件夹下，_config.yml的配置：

  4. 回到命令行部署到远端

    • hexo d
      • 如果：中间会让输入github密码，输入即可，后续部署完成，即可访问你的远端地址：https://humein.github.io/
      • 如果：发现https://humein.github.io/ 和本地的不一样；再次hexo clean -> hexo d 就行了
  5. Blog 更新

    • source/_posts 中更新md文档

    • Blog目录下 执行命令 hexo d 就行了。文件会默认Push到git上面
